### V016-006 配布ファイルと展開後サイズをもう少し減らしたい
- 種類: request
- 優先度: P2
- 発生日: 2026-03-30
- 送信元: 口頭メモ
- 状態: investigating
- 影響範囲: 配布サイズ, ダウンロード負荷, ストレージ使用量, 配布体験
- 現象:
  - zip 圧縮後で 150MB 超、展開後で 300MB 超になってきている
  - Electron アプリとして極端に巨大ではないが、削減余地があるなら詰めたい
- メモ:
  - 手元の `out` 実測では `MMD modoki-win32-x64-0.1.6.zip` が約 161.7MB、展開後 `MMD modoki-win32-x64` が約 372.8MB
  - `forge.config.ts` では `asar: true` は有効
  - 次に確認したいのは Electron 本体、Babylon 系依存、locales、未使用アセット、配布対象ファイルの絞り込み
  - まずは「どこが効いているか」の内訳確認から始める

### V016-007 `.x` アクセサリの半透明影がムラ状に抜ける
- 種類: bug
- 優先度: P2
- 発生日: 2026-04-03
- 送信元: 口頭メモ
- 状態: investigating
- 影響範囲: `.x` アクセサリ, 半透明材質, shadow map, 遠景表示
- 現象:
  - 駅屋根のような半透明アクセサリで、影が均一な半透明ではなく模様っぽくムラ状に抜けて見える
- メモ:
  - 現状の `.x` 側は `alpha blend` ベースの表示を優先している
  - 一方で shadow 側は transparent shadow を有効にしており、texture alpha や面の重なりがそのまま影に乗っている可能性が高い
  - 木のような cutout 系を壊さず汎用的に直すのが難しく、現時点では個別材質ハックは入れない方針
  - 透明影の扱いを材質分類ごとに分けるか、shadow 経路を整理する余地がある
