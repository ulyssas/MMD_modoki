# 選択ボーン / カメラ回転量オーバーレイメモ

作成日: 2026-04-20

対象:
- `src/timeline.ts`
- `src/ui-controller.ts`

追記 2026-04-20:
- 初期案は選択ボーン中心だったが、実装はカメラトラックも同じ回転実測値オーバーレイ対象に含める
- カメラも `X / Y / Z` 回転を同じ色規約で描き、タイムライン上ではボーンと同じ読み方にそろえる

## 1. 背景

次に着手する本命は、補間曲線編集とキー編集。

ただし、その前段として
- 補間が結果としてどう効いているか
- 回転量がどのくらい変化しているか

をタイムライン上で読めると、編集時の判断がかなりしやすくなる。

下パネルの補間欄は「曲線そのものを編集する場所」とし、
タイムライン側は「結果の回転推移を見る場所」として役割を分けたい。

## 2. 基本方針

### 2-1. タイムライン上では補間曲線そのものは出さない

補間欄はすでに下パネルにあるため、同じ情報をタイムライン側に重ねる必要は薄い。
タイムライン上では、補間後に実際どういう回転値になっているかを見る。

### 2-2. キー位置とキー間で見るもの

- キー位置:
  - そのキーに入っている回転値
- キー間:
  - 補間後に評価された実際の回転値

このため、表示は「実測値オーバーレイ」として扱う。

### 2-3. 対象はまず選択中トラックだけ

常時全体表示ではなく、まずは選択中トラックだけに絞る。

理由:
- 視認性を保ちやすい
- 実装範囲を抑えやすい
- 補間編集の足場としてはそれで十分効果がある

## 3. 表示イメージ

### 3-1. 基本形

選択中トラックの行を通常行より高くし、その中に回転の推移を描く。

構成:
- `X / Y / Z` の 3 本線を重ねる
- 既存のキーダイヤはそのまま残す
- 回転線はキーの背後に控えめに置く

### 3-2. 選択行だけ縦に広げる

- 選択中トラックだけ縦幅を広げる
- 非選択トラックは現状維持
- ラベル列も同じ行だけ同じ高さで追従する

別帯を作るのではなく、タイムラインの行の中で完結させる。

### 3-3. XYZ の色は既存 UI とそろえる

新規に色を決めず、既存の軸色をそのまま使う。

- `X`: gizmo / 補間欄の X と同じ
- `Y`: gizmo / 補間欄の Y と同じ
- `Z`: gizmo / 補間欄の Z と同じ

これにより、タイムライン上でも軸の認知をそのまま持ち込める。

## 4. 回転表現の扱い

### 4-1. `180 / -180` またぎ

`180 / -180` またぎは、0 度を横切る直線では結ばない。

今回は「連続角としてきれいに見せる」より、
「その場で評価されたオイラー角がこう見える」を優先する。

そのため、`+180 -> -180` や `-180 -> +180` のまたぎでは、
グラフを上下端へ逃がして一度分断し、反対側から入り直す描画にする。

意図:
- 実際には 0 度を通っていない区間を、0 度経由の直線に見せない
- `180 / -180` のラップで起きた不連続として読み取りやすくする
- unwrap や連続角補正までは入れず、軽い描画ルールで不正確さだけ避ける

### 4-2. 縦スケール

固定 `±180` にすると小さい変化が潰れやすい。
そのため、まずは選択トラック内の最大絶対値に合わせる動的スケールを使う。

## 5. 実装スコープ

### 5-1. 実装済み

- 選択中トラックの回転実測値オーバーレイ
- 対象カテゴリ:
  - `root`
  - `semi-standard`
  - `bone`
  - `camera`
- 選択行のみの縦幅拡張
- ラベル列の同高さ追従
- タイムライン本体の背景無彩色化
- 回転線の不透明度調整

### 5-2. 未対応

- モーフの `0..1` グラフ
- 全トラック同時オーバーレイ
- 複数選択対応
- unwrap や連続角補正
- 表示 ON/OFF スイッチ

## 6. 実装方法

### 6-1. 責務分担

- `src/ui-controller.ts`
  - 選択中トラック変更時
  - キーフレーム再読込時
  に、選択トラック用の回転サンプル列を再生成して `Timeline` に渡す
- `src/timeline.ts`
  - 渡されたサンプル列を、選択行の拡張された高さの中へ折れ線として描画する

要するに:
- 補間評価とサンプル生成は `UIController`
- 描画だけ `Timeline`

で分離している。

### 6-2. サンプル生成

- ボーン:
  - `sampleMovableBonePoseFromTrack()`
  - `sampleBonePoseFromTrack()`
  を再利用
- カメラ:
  - `sampleCameraPoseFromTrack()`
  を再利用
- サンプル範囲:
  - そのトラックの先頭キーから末尾キーまで
- サンプル粒度:
  - 1 フレーム刻み

タイムライン側では独自に補間計算を持たない。

### 6-3. 描画ルール

- `X / Y / Z` は 3 本の細い折れ線で重ね描き
- 色は `--axis-x-color / --axis-y-color / --axis-z-color`
- 線の不透明度は `0.6`
- タイムライン本体の行背景は無彩色ダークグレー
- ラベル列のカテゴリ色分けは維持
- キーダイヤはそのまま前景で残す
- `180 / -180` またぎは上下端で線を分断して描く

## 7. 今回入れていない判断

### 7-1. モーフグラフは一旦入れない

モーフにも `0..1` のグラフを載せる案はあるが、今回は見送る。

理由:
- 今回の主目的は補間曲線編集前の回転読み取り支援
- モーフは回転ほど「結果が読みにくい」問題が強くない
- 下パネル側に既存の値スライダーがあり、当面はそれで足りる

### 7-2. 今は広げすぎない

ここでモーフや高機能表示まで広げると、
補間編集前の足場づくりから少し逸れる。

まずは
- ボーン回転
- カメラ回転

の読み取り支援として固める。
