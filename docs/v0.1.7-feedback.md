## 詳細

### V017-001 カメラモーション読み込み時に位置やタイミングがズレる
- 種類: bug
- 優先度: P1
- 発生日: 2026-04-13
- 送信元: ユーザー報告
- 状態: investigating
- 影響範囲: カメラ VMD 読み込み, 再生, フレームシーク, 最終フレーム停止, タイムライン表示
- 現象:
  - カメラモーションを読み込んだとき、想定したカメラ位置やタイミングからズレるという報告が複数ある
  - 特に最終フレーム付近でズレや評価不整合が出ている可能性がある
- メモ:
  - まずは同じ VMD を本家 MMD と MMD_modoki で比較し、先頭フレーム、中間キー、最終キー、最終フレーム後停止時のカメラ状態を確認する
  - `duration` の扱い、`currentFrame` の丸め、`lastFrame` の決定、最終キーと再生停止フレームの off-by-one を疑う
  - カメラ VMD 読み込み直後、タイムラインシーク時、再生終了時で `MmdCamera -> ArcRotateCamera` 同期タイミングが異なる可能性も確認する
  - v0.1.3 の `V013-009` はカメラキーフレーム登録経路の修正であり、今回は外部カメラモーション読み込み / 評価の別件として扱う
- 2026-04-18 調査メモ:
  - 音声なし再生では、手動フレーム進行と `mmdRuntime.seekAnimation()` が `scene.render()` の後に走っており、表示カメラが 1 描画分遅れる可能性があった
  - 停止中の `seekTo()` は `MmdCamera` を評価していたが、`情報 > 対象 = Camera` のときでも `ArcRotateCamera` への即時同期が `onBeforeRender` / UI サンプル経路に依存していた
  - `src/mmd-manager.ts` で音声なし再生の手動進行を描画前へ移し、`seekTo()` 後に必要な場合だけ `MmdCamera -> ArcRotateCamera` を即時同期するよう修正
  - `src/assets/motion-asset-service.ts` のカメラ VMD 読み込み後の先頭フレーム移動を直接代入ではなく `seekTo(0)` 経由へ寄せた
  - 実機 VMD 比較は未実施のため、状態は `investigating` のまま維持する

### V017-005 カメラが原点から離れると近接モデル材質や床が欠けやすい
- 種類: bug
- 優先度: P1
- 発生日: 2026-04-13
- 送信元: ユーザー報告
- 状態: investigating
- 影響範囲: カメラ表示, viewport, clipping, frustum culling, 床表示, モデル材質描画
- 現象:
  - カメラが原点から離れた状態で、カメラに近接したモデルの材質が消えやすい
  - 同じ条件で床も欠けやすい
- メモ:
  - v0.1.6 の `V016-004`「画面端でポリゴンが欠ける」と関連する可能性はあるが、今回は「原点から離れたカメラ位置」「近接モデル材質」「床欠け」を主条件として別件管理する
  - `minZ / maxZ`、near clip、far clip、projection precision、frustum culling、bounding info 更新を切り分ける
  - 床 mesh の bounds、camera target / radius、MMD カメラ同期時の position / target / distance の意味ずれも確認する
  - 再現時はカメラ座標、target、distance、FOV、WebGPU / WebGL2 経路、床表示 ON/OFF、対象モデルの材質名を記録する
  - 2026-04-24 時点でも、特定ステージで浅い角度や距離変化時に床面が画角内で欠ける報告が継続している
  - `camera.minZ` 調整、PMX mesh の `alwaysSelectAsActiveMesh` / `refreshBoundingInfo`、helper ground の沈め、`dispatchAllSubMeshesOfActiveMeshes` は決定打にならなかった
  - 透明材質経路の影響も疑って loader の alpha path を一時変更したが、キャラモデル側の半透明表示を壊したため撤回した
  - ビルド前時点では既知問題として保留し、コアなモデル透過や通常表示を優先してこれ以上の runtime 変更は入れない

### V017-006 DoF の深度マスクがカラーとずれて見える
- 種類: bug
- 優先度: P2
- 発生日: 2026-04-18
- 送信元: ユーザー報告（スクリーンショット）
- 状態: fixed
- 影響範囲: DoF, depth renderer, viewport aspect resize, 出力プレビュー
- 現象:
  - DoF 適用時にモデル輪郭付近で、カラー画像とぼかし / 深度マスクがずれて見える
- メモ:
  - `DefaultRenderingPipeline.depthOfField` は depth renderer の depth map を使うため、render buffer と depth map のサイズ差があるとマスク位置がずれる可能性がある
  - `LayoutUiController` が viewport aspect に合わせて canvas CSS size を変えるため、engine resize 時に depth map も追従させる必要がある
  - 深度 texture の線形サンプリングは silhouette で depth がにじみ、モデル輪郭の matte ずれに見えやすい
  - 2026-04-18 に DoF depth renderer を nearest sampling にし、engine resize 時に depth map を resize する修正を入れた
  - ユーザー実機確認で DoF のずれが改善したため fixed とする
  - まだ別条件で輪郭にじみが残る場合は foreground/background merge の edge bleed 対策を別件で検討する

### V017-007 プロジェクト保存 / 読み込みで状態が元通り復帰しない
- 種類: bug
- 優先度: P1
- 発生日: 2026-04-18
- 送信元: ユーザー報告
- 状態: investigating
- 影響範囲: project 保存 / 読み込み, 音声, カメラモーション, 照明, DoF, LUT, SSAO, Bloom / tone mapping, 出力設定
- 現象:
  - プロジェクト保存後に読み込み直しても、元の状態へ完全に戻らない
  - 音声ファイル、カメラモーション、照明、ポストエフェクトの復元が特に怪しい
- 2026-04-18 初期調査メモ:
  - 保存側は `keyframes.cameraAnimation` を持つが、読み込み側が存在しない `applyCameraAnimation` を呼んでおり、埋め込みカメラトラック復元で失敗する可能性があった
  - 音声差し替え / project clear 時に runtime から古い audio player を外す前に dispose しており、音声なしプロジェクトへ切り替えたときも runtime 側に古い参照が残る可能性があった
  - 音声メタデータ読み込み後の duration 変化がタイムライン総フレームへ反映されていなかった
  - 保存している SSAO 設定を読み込み時にすべて無効化していた
  - LUT の有効状態 / 強度、contrast、DoF の blur level / near suppression / focal length / lens distortion などが保存または復元から漏れていた
- 2026-04-18 初期修正:
  - `MmdManager.applyCameraAnimation()` を追加し、project 埋め込みカメラトラックを runtime camera へ復元できるようにした
  - audio player の差し替え / project clear / dispose 時に `mmdRuntime.setAudioPlayer(null)` を通すようにした
  - audio duration 変更時に project / 通常読み込みの総フレームを更新するようにした
  - SSAO / LUT / contrast / DoF の一部未復元項目を project schema と import に追加した
  - unit test で SSAO / LUT / DoF / contrast 復元と埋め込みカメラトラック復元を確認する
- 残確認:
  - 実機で、PMX + VMD + カメラ VMD + 音声 + 背景 + 照明 + DoF / LUT / Bloom / Fog を保存して再読み込みする round-trip 確認が必要
  - まだ戻らない項目があれば、保存スキーマ漏れか UI refresh 漏れか runtime 適用漏れかを分けて追加対応する
