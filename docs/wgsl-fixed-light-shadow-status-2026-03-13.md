# fixed light / shadow 系 WGSL プリセット状況メモ

更新日: 2026-03-13

## 概要

2026-03-13 時点で、`MMD Standard` の明暗処理を材質単位で強制したい目的で複数の built-in WGSL プリセットを試した。

対象:

- `full_light`
- `full_light_add`
- `full_shadow`
- `light_and_shadow`

このメモは、それぞれの狙い、現状の扱い、残っている制約を整理するためのもの。

## プリセットごとの位置づけ

### `full_light`

目的:

- PMX 側の toon / 影なし条件を無視して、材質を常に明面寄りで出す
- `光色 > 100%` の加算も受けさせる

現状:

- 髪・服・アクセサリでは用途あり
- 顔のような多層透明レイヤー材質では期待どおりに揃わない

理由:

- 顔は `body_skin` / `face00` / `face01` / `cheek` / eye overlay などの複数レイヤーで合成される
- 各レイヤーで加算しても、後段の alpha ブレンドで見え方が変わる
- 顔テクスチャ自体の描き込みも強く残る

### `full_light_add`

目的:

- 既存の後段 flat light 処理に乗せるのではなく、照明欄の `光色` 値を直接読んで材質内で独自加算する

現状:

- 顔材質の問題解決には至らなかった
- 実装は残しているが、UI からは非表示

判断:

- 実験としては有益だったが、常用プリセットとしては優先度が低い

### `full_shadow`

目的:

- PMX 側の toon / 影なし条件を無視して、材質を常に影面寄りで出す

現状:

- 「影色ベースの簡易固定影」としては使える
- `MMD Standard` の影色と完全一致する fixed shadow にはなっていない

理由:

- `MMD Standard` 側は `shadowTint` だけでなく `toonRaw`、`Toon影響度`、`shadowMask` など複数要素で最終影色が決まる
- それを `@apply-without-toon` 差し込みだけで完全コピーしようとすると不安定化し、黒画や WebGPU validation error が出た

判断:

- 完全一致を狙うより、簡易固定影として割り切る方が安全

### `light_and_shadow`

目的:

- `full_light` / `full_shadow` のような固定面ではなく
- 標準の明暗バンドを PMX の条件に関係なく通す
- 「影の落ちない材質にも明暗を付ける」方向へ寄せる

現状:

- 最初の版は no-toon 経路で不安定で黒画を起こした
- 修正版では `shadow` ベースの安定版にして、黒画は回避
- 完全再現ではないが、見た目としては実用寄り

判断:

- 今回の fixed light / shadow 系では最も実用寄り
- `MMD Standard` の完全コピーではない

## 今回わかった制約

### 1. 顔材質は単純な材質 WGSL だけでは揃いにくい

理由:

- 多層透明レイヤー
- テクスチャに焼き込まれた陰影
- 後段 alpha ブレンド

このため、髪や服と同じロジックでも顔だけ見え方が違いやすい。

### 2. `@apply-without-toon` には差し込み位置由来の制約がある

特に no-toon 経路では、標準 toon 計算で使っている値をそのまま参照できるとは限らない。

今回の実例:

- `toonNdl` を直接使った版は no-toon 経路で不安定化した

### 3. `MMD Standard` の完全コピーはコストが高い

影色や明暗バンドは見た目以上に多くの条件に依存している。
材質ごとの WGSL 差し替えだけで完全一致を狙うと、安定性を落としやすい。

## UI 上の扱い

- `full_light` は表示
- `full_shadow` は表示
- `light_and_shadow` は表示
- `full_light_add` は UI から非表示

## 用語メモ

### `shadowMask=1.0 固定`

本来のシェーダーでは「このピクセルがどれくらい影面か」を計算している。
それを無視して「常に影面」とみなす考え方。

### `toonRaw`

toon テクスチャから取得した影段の生の色。
`MMD Standard` では、これと `shadowTint` を混ぜて影色を作っている。

### `shadowTint`

照明欄の `影色` に相当する値。

### `Toon影響度`

照明欄の `Toon影響度` フェーダー。
`shadowTint` と `toonRaw` のどちらをどれだけ強く使うかに効く。

### `alpha ブレンド`

半透明材質を後ろから順に重ねる処理。
顔材質が揃いにくい主因の一つ。

### `@apply-without-toon`

外部 WGSL を no-toon 材質にも適用するための差し込み運用。
便利だが、標準 toon 計算の値をそのまま使えるとは限らない。

## 現時点の結論

- 顔まで含めて `MMD Standard` と完全に同じ fixed light / fixed shadow を、材質単位 WGSL 差し替えだけで作るのは難しい
- fixed 系プリセットは look-dev 用の簡易プリセットとして割り切るのが妥当
- 顔まで統一したい場合は、材質 WGSL ではなくポストエフェクトや別の合成処理が本命
