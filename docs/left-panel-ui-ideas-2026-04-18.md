# 左パネル UI 案メモ

更新日: 2026-04-18

対象:
- `index.html`
- `src/ui-controller.ts`

## 1. 目的

左パネルを「なんでも置く場所」ではなく、時間軸ナビゲーションの中心として再整理する。

当面の役割は次の通り。

- 左パネル = 時間軸ナビゲーション
- 下パネル = キー編集 / 値編集
- 上パネル = アプリ共通操作

## 2. 先に決めておく方針

### 2-1. Start / Stop は共通範囲

`Start / Stop` は別用途に分けず、共通範囲として扱う。

共通で参照する対象:

- 再生範囲
- ループ範囲
- PNG 連番出力範囲
- WebM 出力範囲

想定フロー:

1. `Start / Stop` を決める
2. その範囲で再生確認する
3. 問題なければそのまま出力する

### 2-2. 出力欄そのものは左パネルへ寄せない

左パネルに入れるのは範囲と時間操作までに留める。

左パネルに置くもの:

- 現在フレーム
- `Start / Stop`
- `Loop`
- 前後フレーム移動
- 前後キー移動
- タイムライン
- 波形表示

出力欄に残すもの:

- 出力形式
- 解像度
- FPS
- codec
- 音声有無

整理:

- 範囲は左で決める
- 出力条件は出力欄で決める

### 2-3. キー登録は下パネルへ寄せる案を本命とする

左パネルは時間軸の確認と移動に集中させたい。

そのため、キー登録・削除・補間・コピー系は下パネルへ寄せる前提で考える。

### 2-4. `Loop` は欲しくなってから入れる

`Loop` 自体は MMD 寄せの機能として候補に残すが、現時点では必須ではない。

現状は人間が再生ボタンを押し直せば足りる場面が多く、今の左パネル UI はようやく責務が整理されてきた段階なので、ここで再生状態を増やして複雑化させない。

判断:

- `Loop` は候補として保持
- ただし実運用で不足が出るまでは未実装でよい
- 左パネルの密度や状態管理を増やさないことを優先する

## 3. 左パネルの最小構成

時間軸 UI として最低限必要なのは次の 4 つ。

1. 現在フレーム
2. `Start`
3. `Stop`
4. タイムライン

その次に足す候補:

5. `Loop`
6. 前後フレーム移動
7. 前後キー移動

タイムライン上では最低限、

- 再生ヘッド
- `Start` 線
- `Stop` 線

が見える必要がある。

## 4. 優先度

### P0: 先に入れる

#### 1. Start / Stop

- 現在フレームを `Start` にセット
- 現在フレームを `Stop` にセット
- `Jump Start / Jump Stop`

#### 2. Loop

- `Start / Stop` 範囲をループ再生

#### 3. Next / Prev Key

- `,` : 前のキーへ
- `.` : 次のキーへ

本命仕様は `選択中トラック基準の次 / 前キー`。

### P1: 次に入れる

#### 4. Waveform

用途:

- サビ頭の把握
- 間奏終わりの把握
- `Start / Stop` 決め
- 音とキーフレ位置の対応確認

実装方針:

- 読み込み時にピーク配列を前計算する
- 重い逐次解析は避ける
- `timeline.ts` の背景レイヤで描く

#### 5. Zoom to Range

- `Start / Stop` 範囲へタイムライン表示を寄せる

#### 6. Playhead Follow

- 再生中にタイムライン表示を再生ヘッドへ追従させる

### P2: 効果は大きいがやや重い

#### 7. Marquee Select

- タイムライン上の複数キーフレーム矩形選択

#### 8. Markers

- サビ頭
- カメラ切替点
- メモ位置

#### 9. Snap

- フレーム
- キー
- `Start / Stop`

への吸着

### P3: 後からでよい

#### 10. Track Collapse / Fold

- ボーン群 / カテゴリ折りたたみ

#### 11. Pin Track

- カメラ、センター、注目トラック固定表示

#### 12. Selection Info

- 選択キー数
- 始点 / 終点
- 範囲長

#### 13. Auto-scroll While Dragging

- タイムライン端ドラッグ時の自動スクロール

### 後回し

#### Ripple Edit 系

動画編集では有効だが、MMD のキーフレーム編集では副作用が大きい。
当面は対象外でよい。

## 7. 本家 MMD から取り込む考え方

本家 MMD は左側にかなり多機能を詰めているが、MMD_modoki ではそのまま密度を真似しない。

参考にする点:

- フレーム操作
- タイムライン
- 範囲指定
- キー移動
- 再生
- 音量や波形のような時間軸補助

そのまま持ち込まない点:

- 画面密度の高さ
- 同機能の重複配置
- 左パネルに値編集まで押し込む構成

## 8. 複数選択

複数選択には 2 種類ある。

1. 対象選択
   - 複数ボーン
   - 複数モーフ
   - 複数トラック
2. キー選択
   - タイムライン上の複数キーフレーム
   - 矩形選択
   - 範囲選択

最初に優先すべきは `複数ボーン値編集` より `タイムライン上の複数キー選択`。

実装順:

1. タイムライン上のキー矩形選択
2. 複数キー移動 / 削除
3. 複数キーコピー / ペースト
4. その後に複数ボーン値編集を再検討

## 9. Undo / Redo

配置は上パネルでよい。

ただし実装は重い。

理由:

- 時間軸専用機能ではない
- ボーン / モーフ / カメラ / 照明 / project state をまたぐ
- runtime と UI の両方を戻す必要がある

当面の導入順:

1. タイムライン編集限定の undo/redo
2. 単一カテゴリ限定 undo/redo
3. 横断 undo/redo

## 10. 保留事項

### 10-1. viewport 直下 UI

別件として、viewport 直下に

- gizmo と連動する操作値パネル
- 本家 MMD で viewport に重なっていたコントローラの受け皿

を置く案がある。

これは左パネルとは別軸の話として扱う。

整理すると、

- viewport 直下 = 現在値操作面
- 下パネル = キー編集面

になりうる。

### 10-2. レイアウト細部

左パネル内レイアウトはまだ確定していない。

候補:

- A: 再生欄の下に `Start / Stop` を置く
- B: 再生欄を 2 ブロック化する
- C: `現在フレーム` ミニセクションを分ける

先に決めるべきなのは配置より責務分担。

## 11. 現時点の結論

左パネルは、まず `Start / Stop / Loop / 次前キー / タイムライン` を中心にした時間軸ナビゲーションとして整理する。

その上で、

- 波形
- 矩形選択
- マーカー

を段階的に足していく。

キー登録や詳細値編集は左へ持ち込まず、下パネルへ寄せる方が整理しやすい。

## 2026-04-19 実装反映メモ

今日の作業で、左パネルと再生欄まわりは以下まで実装済み。

### 左パネル再生欄

- `Start / Stop` の数値入力を左パネル再生欄へ移動
- 文言を `スタート / ストップ` ではなく、UI 表示上は `Start / Stop` 系として整理
- `フレ・スタート` / `フレ・ストップ` のチェックを追加
- 現在の再生仕様:
  - `フレ・スタート ON`: 指定 `Start` フレームから再生
  - `フレ・スタート OFF`: 現フレームから再生
  - `フレ・ストップ ON`: 指定 `Stop` フレームで停止
  - `フレ・ストップ OFF`: `Stop` を自動停止条件に使わず、そのまま後ろまで再生
- 再生欄の `■` 停止ボタンは撤去し、再生 / 一時停止に整理

### タイムライン上段

- 現フレーム表示とフレーム移動を波形帯の上段へ移動
- MMD 寄せの並びとして `|<` `.<` `<` `現フレーム` `>` `>.` `>|` を配置
- 現フレーム数は入力欄化し、数値入力でそのフレームへジャンプ可能
- `|<` は常に `0f`、`>|` は読み込み済みモーションの終端へ移動
- `<` / `>` は 1 フレーム移動

### 波形 / 音量 / レイアウト

- タイムライン上端に波形帯を追加
- 波形の x 座標はタイムラインのフレーム目盛りと揃うように実装
- 音量 UI は波形左に小さく寄せ、`%` 表示は非表示化
- 波形帯、フレーム移動帯、タイムライン本体の縦余白を段ごとに調整
- ラベル列幅も見直し、左端の縦ラインが揃うように調整

### タイムライン欄の役割整理

- 選択ボーン名の文字列表示は削除
- `登録 / 削除` ボタンは出力欄から外し、左パネルのタイムライン欄下へ移動
- 左パネルは時間操作とタイムライン閲覧を主担当に寄せた

### 出力欄との関係

- 出力欄に `再生範囲を使う` チェックを追加
- `OFF`: 従来どおり全フレーム出力
- `ON`: 左パネルの `Start / Stop` 数値範囲を使って出力
- 出力は `フレ・スタート / フレ・ストップ` チェックの状態を見ず、数値範囲だけ参照する

### 書き出し中の進捗表示

- PNG 連番 / WebM ともに、進捗表示が出力範囲の `startFrame / endFrame` を把握できるように変更
- 左パネルで範囲を切っている場合でも、書き出し中のフレーム表示がその範囲前提で読める状態に寄せた

### 現時点の所感

- 左パネルの責務分離はかなり進んだ
- MMD そのままの複製ではなく、MMD の操作感を参照しつつ `modoki` 側で理解しやすい責務分担にしている
- まだ微調整余地はあるが、少なくとも「再生欄」「波形」「フレーム移動」「出力範囲」の導線は一度つながった
