# モデル透過の調査メモ

更新日: 2026-03-27

## 目的
MMD 本家では透過して見えるのに、`MMD modoki` 側では不透明に見える材質がある。その理由を、PMX の元データとローダーの挙動から切り分ける。

## まず分かったこと
現時点で確認できた `頬紅` / `口紅上` / `口紅下` は、PMX 原データ上では透過材質ではなかった。

確認ログ:
- [localhost-1774591474320.log](file:///d:/MMD/modoki/log/localhost-1774591474320.log)

該当材質の主な値:
- `diffuseRGBA = [1, 1, 1, 1]`
- `alpha = 1`
- `transparencyMode = opaque`
- `pmxFlags = 0xe`
- `pmxFlagsDecoded = ["groundShadow", "drawShadow", "receiveShadow"]`
- `diffuseHasAlpha = false`
- `hasOpacityTexture = false`

つまり、こちらが読めている範囲では、
- 材質 alpha では透過していない
- テクスチャ alpha でも透過していない
- PMX フラグでも透過指定は入っていない

## `0xe` の意味
`0xe` は透過フラグではなく、影まわりのフラグだった。

- `groundShadow`
- `drawShadow`
- `receiveShadow`

なので、この値だけでは「透過材質」とは言えない。

## いまの解釈
このモデルの該当材質については、PMX 原データだけを見ると透明にする根拠がない。

考えられる可能性は次のどれか。

1. MMD 本家側が別の要因で透けて見えている
2. 別バージョンのモデル、または別材質モーフを見ている
3. 透過ではなく、色やライティングの見え方でそう見えている
4. こちらのローダーが拾っていない別設定がある

## 今回の結論
現時点では、`頬紅` / `口紅上` / `口紅下` は「透過材質」としては扱えない。

`Alpha Mask` や `alpha-blend` を当てても効かなかったのは自然で、元データに alpha がない材質へ透過処理だけ乗せていたため、見た目が崩れた。

## 今後の確認候補
1. MMD 本家の材質設定画面と、この PMX の同名材質を突き合わせる
2. 材質モーフや別モデル版が使われていないか確認する
3. もし本当に透過している版があるなら、その版の PMX を比較する

## 付記
ローダー側では、透過系プリセットは `alpha` を持つテクスチャにのみ適用する方針にしている。
これは、`hasAlpha=false` の材質に無理に透過設定を当てて、黒残りや変な抜け方を起こすのを避けるため。
